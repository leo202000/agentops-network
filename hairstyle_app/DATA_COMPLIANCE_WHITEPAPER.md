## 📋 执行摘要

本白皮书详细说明 AI 发型生成系统的数据保护措施，确保符合《中华人民共和国个人信息保护法》(PIPL) 和相关法规要求。

**核心承诺**：
- ✅ 用户原图 24 小时自动删除
- ✅ 不用于 AI 训练（除非用户授权）
- ✅ 不对外公开（除非用户同意）
- ✅ 全流程加密保护

## 一、数据生命周期管理

### 1.1 数据流转图

```
用户上传 ──→ 加密传输 ──→ TOS 临时存储 ──→ 火山 API 处理
                                              ↓
用户接收 ←── 永久 TOS 存储 ←── 本地临时 ←── 生成结果
    ↓
24 小时后自动删除原图
```

### 1.2 各阶段保护措施

| 阶段 | 数据 | 存储位置 | 保护措施 | 保留期限 |
|------|------|---------|---------|---------|
| 上传 | 原图 | 用户设备 | 用户自行保管 | - |
| 传输 | 原图 | HTTPS 通道 | TLS 1.3 加密 | 传输完成即删除 |
| 临时存储 | 原图 | TOS | 私有 bucket + ACL | 24 小时 |
| API 处理 | 原图 | 火山服务器 | 内存处理 | 24 小时 |
| 生成结果 | 新图 | TOS | 公共读（仅用户链接） | 永久 |
| 本地缓存 | 临时文件 | 服务器 | 文件系统权限 600 | 1 小时 |

## 三、TOS 存储数据保护

### 3.1 存储架构

```
TOS Bucket: hairfashon
├── hairstyle/              # 用户原图（24 小时删除）
│   ├── 1234567_photo.jpg
│   └── ...
└── hairstyle/results/      # 生成结果（永久保存）
    ├── 1234567_result.jpg
    └── ...
```

### 3.2 访问控制

**Bucket 策略**：
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {"AWS": ["*"]},
      "Action": ["tos:GetObject"],
      "Resource": ["arn:tos:*:*:hairfashon/hairstyle/results/*"],
      "Condition": {
        "StringLike": {"tos:prefix": "hairstyle/results/"}
      }
    }
  ]
}
```

**说明**：
- `hairstyle/` 目录：私有（仅服务可访问）
- `hairstyle/results/` 目录：公共读（仅用户可获取链接）

### 3.3 自动删除机制

**清理脚本**：`cleanup_tos_data.py`

**执行时间**：每天凌晨 2 点

**清理规则**：
```python
# 删除超过 24 小时的原图
if file.created_at < (now - 24 hours) and 'results/' not in file.path:
    delete(file)
```

**日志记录**：
```
[2026-03-22 02:00:01] 开始清理过期数据
[2026-03-22 02:00:15] ✅ 删除：hairstyle/1234567_photo.jpg
[2026-03-22 02:00:16] ✅ 删除：hairstyle/1234568_photo.jpg
[2026-03-22 02:00:20] 清理完成：删除 15 个文件，保留 150 个文件
```

## 四、用户授权管理

### 4.1 授权流程

```
用户注册 ──→ 展示隐私协议 ──→ 用户勾选同意 ──→ 记录同意时间
                                                      ↓
                                              存储同意记录（90 天）
```

### 4.2 同意记录

**存储内容**：
```json
{
  "user_id": "user_12345",
  "agreement_version": "1.0.0",
  "agreed_at": "2026-03-22T14:30:00+08:00",
  "ip_address": "192.168.1.100",
  "user_agent": "Mozilla/5.0...",
  "consent_items": [
    "data_collection",
    "data_processing",
    "third_party_transfer",
    "auto_deletion"
  ]
}
```

### 4.3 授权撤回

**用户可随时撤回同意**：

- 方式：联系客服 / 发送邮件
- 处理时限：7 个工作日
- 后果：停止数据处理，删除已有数据

## 五、数据安全事件应急

### 5.1 事件分级

| 级别 | 定义 | 响应时限 |
|------|------|---------|
| 特别重大 | >100 万用户数据泄露 | 立即 |
| 重大 | 10-100 万用户数据泄露 | 1 小时 |
| 较大 | 1-10 万用户数据泄露 | 4 小时 |
| 一般 | <1 万用户数据泄露 | 24 小时 |

### 5.2 响应流程

```
发现事件 ──→ 初步评估 ──→ 启动应急预案 ──→ 通知用户
                                           ↓
                                      报告监管部门
                                           ↓
                                      调查原因 ──→ 整改
```
