### 问题 2：API 并发超限

#### 症状
- 错误信息：`Request Has Reached API Concurrent Limit, Please Try Later`
- 需要等待并发释放后重试

#### 根本原因
- 火山引擎 API 有**并发请求数限制**
- 可能同时发送了多个请求
- 超过了账户的并发配额

#### 并发限制分析

| 账户类型 | 可能并发数 | 说明 |
|---------|-----------|------|
| 免费/测试 | 1-5 | 较低并发 |
| 企业账户 | 10-50 | 中等并发 |
| 大客户 | 50+ | 高并发 |

#### 解决方案选项

**选项 A**: 实施请求队列 ⭐ 推荐
```python
# 串行化请求
async def process_with_queue(images):
    for image in images:
        result = submit_task(image)
        await asyncio.sleep(0.5)  # 避免并发
```

**选项 B**: 添加重试机制
```python
# 指数退避重试
@retry(wait=wait_exponential(multiplier=1, min=4, max=10))
def submit_with_retry():
    pass
```

**选项 C**: 联系提升并发配额
```
- 联系火山引擎销售
- 申请提高并发限制
- 适合生产环境
```
