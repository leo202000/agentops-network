## 📋 问题描述

根据即梦工程师反馈：

> 您的提交素材存在问题，入参链接必须是公网可以访问的资源链接，本地的链接是使用不了的，导致服务这边获取素材卡住了，所以还请调整一下资源链接为公网可访问资源。

### ❌ 错误的请求示例

```json
{
  "height": 1024,
  "image_url": "http://localhost:8000/uploads/test_floral_hair.jpg",
  "prompt": "bob cut hairstyle, short hair, realistic photo",
  "req_key": "seed3l_single_ip",
  "width": 1024
}
```

**问题**:
1. ❌ `http://localhost:8000/...` - 本地链接，即梦服务器无法访问
2. ✅ `req_key: seed3l_single_ip` - 这个是正确的

---

## ✅ 解决方案

### 方案 1: 使用 Base64 编码（默认，立即可用）

系统会自动将本地图片转换为 base64 格式：

```python
# 自动转换
image_url = "data:image/jpeg;base64,/9j/4AAQSkZJRg..."
```

**优点**:
- ✅ 无需配置
- ✅ 立即可用
- ✅ 适合测试

**缺点**:
- ⚠️ 图片较大时传输慢

### 方案 2: 使用对象存储（生产环境推荐）

支持火山引擎 TOS 或阿里云 OSS：

```python
# 上传到 TOS
image_url = "https://bucket.tos-cn-beijing.volces.com/hairstyle/xxx.jpg"

# 或上传到 OSS
image_url = "https://bucket.oss-cn-hangzhou.aliyuncs.com/hairstyle/xxx.jpg"
```

**优点**:
- ✅ 公网可访问
- ✅ 传输速度快
- ✅ 适合生产环境

**缺点**:
- ⚠️ 需要配置 Bucket 和密钥

## 🔍 技术细节

### 正确的 API 请求

```python
# 使用 base64
body = {
    "req_key": "seed3l_single_ip",  # ✅ 正确的 req_key
    "image_url": "data:image/jpeg;base64,/9j/4AAQSkZJRg...",  # ✅ base64
    "prompt": "保持人物脸部完全一致，只改变发型为短发，realistic photo",
    "width": 1024,
    "height": 1024
}

# 或使用公网 URL
body = {
    "req_key": "seed3l_single_ip",
    "image_url": "https://bucket.tos-cn-beijing.volces.com/photo.jpg",  # ✅ 公网 URL
    "prompt": "保持人物脸部完全一致，只改变发型为短发，realistic photo",
    "width": 1024,
    "height": 1024
}
```

### 自动上传逻辑

```python
# 1. 检查 TOS 配置
if TOS_BUCKET and TOS_ACCESS_KEY:
    upload_to_tos(image_path)

# 2. 检查 OSS 配置
elif OSS_BUCKET and OSS_ACCESS_KEY_ID:
    upload_to_oss(image_path)

# 3. 默认使用 base64
else:
    convert_to_base64(image_path)
```

### 生产模式（TOS/OSS）

1. 配置环境变量（`.env` 文件）：


