# 发型变换优化指南 v2

**创建日期**: 2026-03-21  
**问题**: 变换不彻底，还留有一段头发未变换  
**解决方案**: 优化参数配置 + 负面提示词

---

## 🔍 问题分析

### 根本原因

根据用户反馈和测试结果，"变换不彻底"问题主要由以下原因导致：

1. **重绘强度 (strength) 过低** ⭐⭐⭐
   - 原默认值：0.6
   - 问题：模型倾向于保留原图特征，导致旧发型痕迹残留
   - 解决：提高到 0.75-0.85

2. **缺少负面提示词 (negative_prompt)** ⭐⭐
   - 原实现：未使用
   - 问题：模型不知道要去除哪些特征
   - 解决：明确指定"short hair, bob cut"等要去除的特征

3. **提示词描述不够详细** ⭐
   - 原实现：简单描述（如"curly hairstyle"）
   - 问题：模型理解模糊，生成效果不精确
   - 解决：详细描述长度、质地、风格

4. **CFG Scale 和 Sample Steps 未优化**
   - 原实现：使用默认值
   - 问题：prompt 遵循度不够高
   - 解决：提高到 9.0-10.0 和 40-45 步

## 📊 参数推荐配置

### 场景 1: 短发 → 长发（大幅度变换）

```json
{
  "strength": 0.85,
  "cfg_scale": 10.0,
  "sample_steps": 45,
  "prompt": "及腰超长直发，waist-length long hair, flowing hair past waist, silky smooth",
  "negative_prompt": "short hair, bob cut, medium hair, shoulder length, 短发，齐肩发，中长发"
}
```

**说明**:
- strength=0.85: 最大变化幅度
- negative_prompt: 明确去除短发特征
- prompt: 强调"及腰"、"超长"等长度关键词

### 场景 2: 直发 → 卷发（中等变换）

```json
{
  "strength": 0.75,
  "cfg_scale": 9.0,
  "sample_steps": 40,
  "prompt": "浓密大波浪卷发，glamorous big wavy curls, voluminous waves",
  "negative_prompt": "straight hair, 直发"
}
```

**说明**:
- strength=0.75: 中等变化，保持质量
- negative_prompt: 只去除直发特征

### 场景 3: 微调（轻微调整）

```json
{
  "strength": 0.55,
  "cfg_scale": 8.0,
  "sample_steps": 35,
  "prompt": "微调发型，natural look",
  "negative_prompt": null
}
```

**说明**:
- strength=0.55: 轻微调整
- 不需要 negative_prompt

## 🎯 最佳实践

### 1. 提示词编写技巧

**✅ 好的提示词**:
```
及腰超长直发，waist-length long hair, flowing hair past waist, silky smooth, 
elegant and graceful, realistic photo, high quality, professional photography
```

**❌ 差的提示词**:
```
长发
```

**技巧**:
- 使用具体长度描述（及腰、过肩、齐肩）
- 添加质地描述（丝滑、浓密、蓬松）
- 添加风格描述（优雅、性感、专业）
- 中英双语描述，提高理解准确度

### 2. 负面提示词编写技巧

**针对短发→长发**:
```
short hair, bob cut, medium hair, shoulder length, 短发，齐肩发，中长发
```

**针对直发→卷发**:
```
straight hair, sleek hair, 直发
```

**针对长发→短发**:
```
long hair, waist length, flowing hair, 长发，及腰发
```

**技巧**:
- 列出所有可能的旧发型特征
- 包括长度、质地、风格
- 中英双语

### 3. 参数调优策略

**逐步提高 strength**:
```
第一次：0.70 (测试)
第二次：0.75 (如果变化不够)
第三次：0.85 (最大变化)
第四次：0.90+ (谨慎使用，可能影响脸部)
```

**观察指标**:
- ✅ 发型变化是否彻底
- ✅ 脸部特征是否保持
- ✅ 图片质量是否清晰
- ⚠️ 如果出现畸变，降低 strength 或 cfg_scale

## ⚠️ 注意事项

### 1. strength 上限

- **推荐范围**: 0.70-0.85
- **最大值**: 0.90（谨慎使用）
- **风险**: 过高可能导致脸部特征变化

### 2. API 限流

- 批量测试时，请求间隔至少 2-3 秒
- 使用 `--compare` 时已自动添加间隔

### 3. 图片质量

- 上传清晰、光线良好的照片
- 避免模糊、过暗、过曝
- 推荐分辨率：1024x1024 或更高

### 4. 模型能力边界

- seed3l_single_ip 主要用于"角色特征保持"
- 超大幅度变换（如光头→及腰长发）可能仍有难度
- 如遇到瓶颈，考虑联系即梦客服获取专业建议

## 🔧 故障排查

### 问题 1: 变换仍然不彻底

**检查清单**:
- [ ] strength 是否 ≥ 0.75
- [ ] negative_prompt 是否包含旧发型特征
- [ ] prompt 是否详细描述新发型
- [ ] cfg_scale 是否 ≥ 9.0

**解决方案**:
1. 提高 strength 到 0.85
2. 扩展 negative_prompt（添加更多旧发型描述）
3. 优化 prompt（更详细的长度、质地描述）

### 问题 2: 脸部特征变化

**原因**: strength 或 cfg_scale 过高

**解决方案**:
1. 降低 strength (0.85 → 0.75)
2. 降低 cfg_scale (10.0 → 9.0)
3. 确保 prompt 包含"保持人物脸部完全一致"

### 问题 3: 图片质量下降

**原因**: sample_steps 过低或 strength 过高

**解决方案**:
1. 提高 sample_steps 到 45-50
2. 适当降低 strength
3. 检查原图质量
