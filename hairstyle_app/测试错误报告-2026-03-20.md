## 🔍 根本原因分析

### 1. API 文档版本混乱

火山引擎即梦 API 有多个版本和端点：

| 端点 | 服务 | 状态 | 用途 |
|------|------|------|------|
| `jimeng-api.volcengineapi.com` | jimeng | ❌ 无法访问 | 即梦独立 API |
| `visual.volcengineapi.com` | cv | ✅ 可访问 | 视觉服务（包含即梦） |

**问题**: 我们使用了错误的端点配置

### 2. 参数格式不一致

**即梦 API 格式**:
```json
{
  "model": "dream-4.0",
  "init_image": "..."
}
```

**视觉服务格式**:
```json
{
  "req_key": "seed3l_single_ip",
  "image_url": "..."
}
```

**问题**: 混用了两种不同的 API 格式

### 3. 同步 vs 异步

- **即梦 API**: 同步返回结果
- **视觉服务**: 异步（提交 → 获取 task_id → 轮询结果）

**问题**: 代码期望同步返回，但实际是异步 API

## ✅ 解决方案

### 方案 1: 使用 visual.volcengineapi.com 端点（推荐）

**修改配置**:
```python
# 端点配置
host = "visual.volcengineapi.com"
service = "cv"
region = "cn-north-1"

# API 参数
query = {
    "Action": "CVSync2AsyncSubmitTask",
    "Version": "2022-08-31"
}

body = {
    "req_key": "seed3l_single_ip",
    "image_url": "...",  # 公网 URL 或 base64
    "prompt": "...",
    "width": 1024,
    "height": 1024,
    "strength": 0.6
}
```

**优点**:
- ✅ 端点可访问
