# 即梦官网抓包指南 - 抓取隐藏参数

**问题**: 官网界面不显示参数值  
**解决**: 用浏览器开发者工具抓取 API 请求

---

## ️ 方法 1: 浏览器开发者工具 (推荐⭐⭐⭐⭐⭐)

### 步骤 1: 打开开发者工具

**Chrome/Edge**:
1. 按 `F12` 键
2. 或按 `Ctrl+Shift+I` (Windows) / `Cmd+Option+I` (Mac)
3. 或右键页面 → "检查" / "Inspect"

**Safari**:
1. 先启用开发菜单：Safari → 偏好设置 → 高级 → 勾选"显示开发菜单"
2. 按 `Cmd+Option+I`
3. 或右键页面 → "检查元素"

---

### 步骤 2: 切换到 Network (网络) 标签

**界面示例**:
```
┌─────────────────────────────────────┐
│ Elements │ Console │ Network │ ... │
├─────────────────────────────────────┤
│                                     │
│  Network (网络)                      │
│  ─────────────────────────────────  │
│  Name          Type    Size    Time │
│  ─────────────────────────────────  │
│  (等待请求出现...)                   │
│                                     │
└─────────────────────────────────────┘
```

**操作**:
1. 点击 "Network" 标签
2. 确保录制按钮是 **红色●** (正在录制)
3. 清空之前的请求 (点击 🚫 图标)

---

### 步骤 3: 开始生成图片

**操作**:
1. 上传照片
2. 输入 prompt: "大波浪卷发"
3. 点击"生成"按钮
4. **同时观察 Network 面板**

---

### 步骤 4: 找到 API 请求

**寻找以下特征**:
- **请求类型**: `POST` 或 `XHR` 或 `Fetch`
- **请求名称**可能包含:
  - `generate`
  - `create`
  - `task`
  - `submit`
  - `image`
  - `volcengine`
  - `jimeng`

**界面示例**:
```
┌─────────────────────────────────────┐
│  Name          Type    Size    Time │
│  ─────────────────────────────────  │
│  generate      fetch   2KB    850ms │ ← 可能是这个!
│  upload        xhr     1KB    320ms │
│  status        xhr     500B   150ms │
└─────────────────────────────────────┘
```

---

### 步骤 5: 查看请求详情 ⭐⭐⭐

**操作**:
1. **点击**找到的请求
2. 查看右侧/下方详情

**需要查看的标签**:

#### A. Headers (请求头)
```
Request URL: https://xxx.volcengineapi.com/xxx
Request Method: POST
Content-Type: application/json
```

#### B. Request Payload / Body (请求体) ⭐⭐⭐
**这是最重要的！包含所有参数！**

可能长这样:
```json
{
  "req_key": "seed3l_single_ip",
  "prompt": "大波浪卷发...",
  "strength": 0.75,
  "cfg_scale": 9.0,
  "sample_steps": 40,
  "width": 1024,
  "height": 1024,
  "seed": -1
}
```

#### C. Response (响应)
```json
{
  "code": 10000,
  "data": {
    "task_id": "xxx",
    "image_url": "..."
  }
}
```

---

### 步骤 6: 复制请求数据

**操作**:
1. 在 Request Payload/Body 区域
2. **右键** → "Copy" → "Copy all as JSON"
3. 或手动截图

**分享给我们**:
```json
{
  "strength": 0.75,
  "cfg_scale": 9.0,
  "sample_steps": 40,
  ...
}
```

## ⚠️ 常见问题

### Q1: Network 面板是空的？
**A**: 
- 确保录制按钮是红色●
- 点击 🚫 清空后重新生成
- 刷新页面再试

### Q2: 太多请求找不到？
**A**:
- 在过滤框输入 "generate" 或 "volc"
- 按 Type 过滤，只看 "fetch" 或 "xhr"
- 看 Size 较大的请求 (通常包含参数)

### Q3: 请求内容是加密的？
**A**:
- 有些网站会加密请求体
- 尝试查看 URL 参数
- 或查看响应内容找线索

### Q4: 找不到 strength/cfg 等参数？
**A**:
- 可能参数名不同 (如 denoising_strength)
- 可能使用默认值不发送
- 截图整个请求体，我们一起分析

### Q5: 请求太多看花眼？
**A**:
1. 清空 Network (🚫 按钮)
2. 只点击"生成"按钮
3. 观察新出现的请求
4. 通常 1-3 个请求
