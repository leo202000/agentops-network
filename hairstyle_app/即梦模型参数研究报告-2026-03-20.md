## 🔍 核心发现

### 发现 1: 人物特征保持 ✅

**结论**: seed3l_single_ip 模型在保持人物脸部特征方面表现优秀

**证据**:
- 所有生成结果中，人物脸部基本保持一致
- 即使 strength 高达 0.8，脸型、五官仍保持不变
- 这是模型的核心优势

### 发现 2: 发型变换不完整 ⚠️

**问题**: 短发→长发的长度变化效果有限

**表现**:
- 羊毛卷：卷发效果可见，但卷度不够浓密
- 大波浪：波浪效果良好，但长度变化不明显
- 及腰长发：长度变化有限，可能仍是短发

**可能原因**:
1. **seed3l_single_ip 模型设计**: 主要用于"角色特征保持"，发型变化幅度受限
2. **strength 参数上限**: 即使设置到 0.8，模型可能仍有内部限制
3. **prompt 权重**: "长发"关键词权重可能不足
4. **模型能力边界**: 可能不支持大幅度发型长度变化

### 发现 3: 参数影响规律

| 参数 | 对发型变化的影响 | 推荐值 |
|------|-----------------|--------|
| **strength** | 影响最大，0.7+ 才有明显变化 | 0.75-0.85 |
| **cfg_scale** | 次要影响，提高 prompt 遵循度 | 8.5-10.0 |
| **sample_steps** | 影响质量，对变化幅度影响小 | 35-45 |

## 🎯 问题分析

### 问题定义

**用户期望**: 短发照片 → 生成明显不同的长发/卷发效果  
**实际结果**: 人物保持完整，但发型变化不够"完整"/"彻底"

### 根本原因假设

#### 假设 1: 模型能力限制 (最可能) ⭐

**seed3l_single_ip** 模型的设计目标：
- ✅ 保持角色特征（脸部）
- ✅ 轻微发型调整（同长度风格变化）
- ❌ 大幅度发型变换（短发→长发）

**证据**:
- 卷发/波浪在同长度内效果较好
- 长度变化（短发→长发）效果有限
- 模型名称中的 "single_ip" 暗示单一角色保持

#### 假设 2: 参数未优化

可能我们的参数组合还不是最优：
- strength 可能需要 0.85+（但风险更高）
- 需要负面 prompt: "no short hair, not bob cut"
- 需要更强的长度描述词

#### 假设 3: 需要特定模型/接口

即梦可能有专门的"发型变换"模型或接口：
- 不同的 req_key
- 支持更大变化幅度的模型

## 💡 优化方案

### 方案 1: 参数优化 (低成本)

#### 推荐参数组合

**高变化幅度** (尝试更大变化):
```json

  "sample_steps": 45,
  "cfg_scale": 10.0,
  "prompt": "及腰长发，超长发披肩，waist-length long hair, flowing long hair, ...",
  "negative_prompt": "short hair, bob cut, medium hair, 短发, 中长发"
}
```

**平衡型** (保持质量):
```json

  "strength": 0.75,

  "cfg_scale": 9.0,
  "prompt": "...",
  "negative_prompt": "short hair, bob cut"
}
```

### 方案 2: 多阶段生成 (中成本)

**思路**: 分步实现短发→长发

```
阶段 1: 短发 → 中长发 (strength 0.6)
阶段 2: 中长发 → 长发 (strength 0.6)
阶段 3: 长发 → 特定发型 (strength 0.5)
```

**优点**: 每步变化较小，累积效果明显  
**缺点**: 需要多次 API 调用，成本高
