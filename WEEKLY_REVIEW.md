# 📅 每周记忆系统审查清单

**频率**: 每周执行一次 (建议：周日晚上)  
**时间**: 约 15-20 分钟

---

## ✅ 审查步骤

### 步骤 1: 技能清单验证 (5 分钟)

```bash
# 1. 运行技能清单更新
./skills/update-skills-list.sh

# 2. 对比实际技能数量
echo "实际技能数:"
ls -d /root/.openclaw/workspace/skills/*/ | wc -l

# 3. 检查 CHANGELOG.md
cat skills/CHANGELOG.md | head -30
```

**检查项**:
- [ ] INSTALLED_SKILLS.md 是否与实际一致？
- [ ] 本周安装的 skill 是否都记录了？
- [ ] CHANGELOG.md 是否更新了？

---

### 步骤 2: 记忆完整性检查 (5 分钟)

```bash
# 1. 检查 MEMORY.md 最后更新
stat -c %y MEMORY.md

# 2. 检查本周日志文件
ls -lt memory/*.md | head -7

# 3. 检查当日日志
cat memory/$(date '+%Y-%m-%d').md
```

**检查项**:
- [ ] MEMORY.md 是否包含本周重要事件？
- [ ] 是否有超过 3 天未更新的日志？
- [ ] 是否有重复或冲突的信息？
- [ ] 敏感信息是否已脱敏？

---

### 步骤 3: 工作进行中检查 (3 分钟)

```bash
# 1. 检查 active_tasks.md
cat memory/working/active_tasks.md

# 2. 检查 key_decisions.md
cat memory/working/key_decisions.md

# 3. 检查 anti-forgetting-lesson.md
cat memory/working/anti-forgetting-lesson.md
```

**检查项**:
- [ ] 进行中的任务是否还有效？
- [ ] 已完成的任务是否标记了？
- [ ] 关键决策是否记录了？
- [ ] 是否有需要升级到 MEMORY.md 的信息？

---

### 步骤 4: 遗忘事件回顾 (3 分钟)

**回顾本周**:
- [ ] 是否有"你忘记了..."的反馈？
- [ ] 是否有重复解决的问题？
- [ ] 是否有遗漏的重要信息？

**如果有**:
1. 记录到 `memory/events/YYYY-MM-DD-forgetting-incident.md`
2. 分析原因
3. 更新防遗忘协议

---

### 步骤 5: 系统优化 (4 分钟)

**检查脚本和工具**:
```bash
# 1. 检查 start-session.sh 是否运行正常
./start-session.sh

# 2. 检查 update-skills-list.sh 是否更新清单
./skills/update-skills-list.sh

# 3. 检查文件权限
ls -la .env MEMORY.md
```

**优化项**:
- [ ] 是否有可以自动化的手动步骤？
- [ ] 是否有可以简化的流程？
- [ ] 是否有需要更新的文档？

## 🔧 常见问题处理

### Q1: 技能清单不一致
**解决**: 
```bash
# 重新生成清单
./skills/update-skills-list.sh

# 手动对比
diff <(ls workspace/skills/) <(cat skills/INSTALLED_SKILLS.md | grep "\`" | cut -d'\`' -f2)
```

### Q2: MEMORY.md 太久未更新
**解决**:
1. 检查本周日志文件
2. 提取重要事件
3. 立即更新 MEMORY.md
4. 设置每周提醒

### Q3: 遗忘事件频繁
**解决**:
1. 分析遗忘原因
2. 检查是否执行了启动检查
3. 加强记忆更新频率
4. 考虑增加自动化
