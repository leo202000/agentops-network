## 🔐 阶段 5: 权限配置

配置技能权限（openclaw.json）：

```json
{
  "skills": {
    "skill-name": {
      "enabled": true,
      "permissions": {
        "exec": false,
        "writeFile": ["./workspace/*"],
        "readFile": ["./workspace/*"],
        "fetch": ["api.trusted.com"]
      }
    }
  }
}
```

- [ ] 权限已限制到最小必要范围
- [ ] exec 权限已禁用（除非必要）
- [ ] 文件访问路径已限制
- [ ] 网络请求域名已白名单

## 🆘 应急计划

如果发现问题：

1. 立即禁用技能
   ```bash
   openclaw skills disable <skill-name>
   ```

2. 删除技能
   ```bash
   openclaw skills remove <skill-name>
   ```

3. 审查审计日志
   ```bash
   openclaw audit log --since "1h"
   ```

4. 更改可能泄露的密钥

5. 报告问题给 ClawHub
