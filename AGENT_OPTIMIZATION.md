# Agent Run 超时优化方案

## 问题诊断

### 现状
- 某些嵌入式 agent run 超过 10 分钟（600 秒）未完成
- 被系统终止，导致 FailoverError
- 触发新 session 重试

### 影响
- 阻塞 agent 资源
- 可能导致排队或请求延迟
- 用户体验差（LLM 响应慢或超时）

---

## 优化策略

### 1. 任务拆分机制

**原则**: 将大任务拆分为多个小任务，每个任务控制在 2-3 分钟内完成

**实施方式**:
```
大任务 → 子任务1 → 子任务2 → 子任务3 → 合并结果
 (10min)   (2min)    (2min)    (2min)     (1min)
```

### 2. 异步处理 + Checkpoint

**原则**: 复杂任务使用异步处理，定期保存进度

**实施方式**:
- 使用 `sessions_spawn` 创建子任务
- 定期保存 checkpoint 到文件
- 超时后可以从 checkpoint 恢复

### 3. 超时监控和告警

**原则**: 实时监控任务执行时间，提前预警

**实施方式**:
- 设置软超时（警告）和硬超时（终止）
- 记录任务执行时间统计
- 超时后自动降级或切换模型

### 4. 模型选择优化

**原则**: 根据任务类型选择合适的模型

**模型配置**:
| 模型 | 类型 | 超时 | 适用场景 |
|------|------|------|----------|
| qwen3.5-plus | 全能 | 30s | 默认首选 |
| kimi-k2.5 | 快速 | 25s | 需要快速响应 |
| MiniMax-M2.5 | 文本 | 20s | 纯文本任务 |
