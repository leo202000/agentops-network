## 🔧 OpenClaw 多 Bot 配置

### 问题发现
- 用户遇到"第二个bot不能匹配"的问题
- 同时提到"本地龙虾不匹配"

### 配置方案
OpenClaw 支持多 bot token，使用 `accounts` 对象配置：

```json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "accounts": {
        "bot1": {
          "botToken": "第一个bot的token",
          "dmPolicy": "pairing",
          "groupPolicy": "allowlist"
        },
        "bot2": {
          "botToken": "第二个bot的token",
          "dmPolicy": "pairing",
          "groupPolicy": "allowlist"
        }
      }
    }
  }
}
```

**关键点**：
1. 顶层 `botToken` → 默认/主 bot（向后兼容）
2. `accounts` 对象 → 多个 bot 配置（独立 token）
3. 每个 bot 有独立的 `dmPolicy`、`groupPolicy`、`allowFrom` 等设置

## ⛏ BOTCOIN 挖矿技能

### 技能来源
- URL: https://agentmoney.net/skill.md
- 名称: botcoin-miner
- 描述: 在 Base 链上通过解决 AI 挑战挖 BOTCOIN

### 前置要求
1. **Bankr API Key**（写权限）
   - 在 https://bankr.bot/api 注册
   - 设置环境变量 `BANKR_API_KEY`
   - 必须启用 Agent API 且关闭只读模式

2. **Bankr Skill 已安装**
   - 需要先安装 Bankr OpenClaw skill

3. **Base 链上的 ETH**
   - 用于 gas（通常 < $0.01）

4. **至少 1,000,000 BOTCOIN**
   - 挖矿的最小要求

### 挖矿流程
1. 认证 → 获取矿工地址
2. 检查余额 → 确保有 BOTCOIN 和 ETH
3. 身份验证握手 → 如果 coordinator 启用认证
4. 挖矿循环：
   - 请求挑战（带唯一 nonce）
   - 解决混合挑战（阅读 25 家虚构公司的文档）
   - 提交答案（返回相同 nonce）
   - 上链发布收据（通过 Bankr）
5. 声明奖励 → epoch 结束后（24 小时）

### 挑战特点
- 阅读长文档并识别实体
- 需要多跳推理和精确计算
- 生成满足所有约束条件的单行答案字符串

### 积分规则（基于 BOTCOIN 余额）
- ≥1,000,000 → 1 积分/解
- ≥10,000,000 → 2 积分/解
- ≥100,000,000 → 3 积分/解
