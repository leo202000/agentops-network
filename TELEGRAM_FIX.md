---

## ❗️ 问题诊断

### 真正原因
```
telegram message failed: (400: Bad Request: message is too long)
```

**翻译**:
- ✅ 机器人已收到消息
- ✅ 尝试回复了
- ❌ 回复内容超过 4096 字符被 Telegram 拒绝
- **表现**: 用户看到"没反应"

---

## 📊 Telegram 限制

| 平台 | 单条消息限制 | 实际使用 |
|------|-------------|---------|
| Telegram Bot | 4096 字符 | 建议 ≤3500 |
| 你的 AI 输出 | 几千字 | ❌ 超长 |
| 结果 | 400: Bad Request | ❌ 被拦截 |

---

## ✅ 修复方案

### 方案：消息分片发送

**核心代码**:
```python
MAX_LEN = 3500  # 留 596 字符余量

def safe_reply(text):
    if len(text) <= MAX_LEN:
        return [text]
    return [text[i:i+MAX_LEN] for i in range(0, len(text), MAX_LEN)]

# 使用
for msg in safe_reply(response):
    send_message(msg)
```

**效果**:
- ✅ 自动分割长消息
- ✅ 保持段落可读性
- ✅ 每条 ≤3500 字符
- ✅ 全部发送成功
